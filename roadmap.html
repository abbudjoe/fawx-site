Roadmap

What's built, what's in progress, and where Fawx is headed. Updated as things ship.

Shipped

Core Engine

The foundation. A single Rust binary that runs an agentic loop with tool calling, memory, and multi-provider support.

  • ✓ 7-step agentic loop with multi-tool calls
  • ✓ 13+ built-in tools (files, web, browser, code execution)
  • ✓ Persistent memory with semantic search (built-in embedding model)
  • ✓ Multi-provider support (Claude, GPT, local models)
  • ✓ Encrypted credential store (AES-256-GCM)
  • ✓ Config management with hot-reload (SIGHUP restart)
  • ✓ Token-by-token streaming over SSE
Shipped

TUI + Multi-Channel

A terminal UI built with ratatui, plus Telegram and HTTP API channels. Same engine, same memory, different surfaces.

  • ✓ Full terminal UI with streaming, markdown rendering, syntax highlighting
  • ✓ Telegram bot channel
  • ✓ HTTP API (headless mode)
  • ✓ Interactive setup wizard (fawx setup)
  • ✓ Model switching mid-conversation
Shipped

Kernel Safety Architecture

Hard separation between the safety kernel and the intelligence layer. The kernel is compiled into the binary and cannot be modified at runtime, even by the agent itself.

  • ✓ Proposal gates for write operations (human approval required)
  • ✓ Tier-3 immutable paths (compiled into binary, never overwritable)
  • ✓ Tool execution sandbox with allowlists
  • ✓ Retry budget and loop guards (prevent runaway agents)
  • ✓ Memory decay and boundary enforcement
Shipped

WASM Skill Plugins

Hot-loadable WebAssembly skills with cryptographic signature verification. The agent can install, load, and use new capabilities without restarting.

  • ✓ WASI-based skill runtime with sandboxing
  • ✓ Cryptographic signature verification (Ed25519)
  • ✓ Hot-reload without restart
  • ✓ Skill creation CLI (fawx skill create)
  • ✓ Marketplace skills: weather, vision, TTS, browser, calculator
Building

Proof of Fitness Protocol

Autonomous experimentation where competing agents solve the same problem with different strategies. Results are scored objectively and recorded in a tamper-evident chain.

  • ✓ Core protocol types, chain, and scoring (fx-consensus crate)
  • ✓ Protocol engine with orchestrator and cross-evaluation
  • ✓ Candidate generators and evaluators (build/test fitness)
  • ✓ CLI: fawx experiment run/chain/show/verify
  • ✓ Real LLM integration with cargo workspace evaluation
  • ✓ Subagent-driven nodes (each node is a full Fawx agent)
  • ✓ Experiment tool wired into the agentic loop
  • ◐ Diagnostics and observability (--detail, --raw)
  • ◐ Agent-aware prompts (subagents read code before patching)
  • ○ Persist all candidate patches in chain
  • ○ TUI side panel with live experiment progress
  • ○ Canonical chain serialization for forward compatibility
Planned

Distributed Fleet

Experiments that run across multiple machines. A node registry discovers available compute, and the experiment runner distributes work across the fleet.

  • ✓ Fleet primitives: Channel trait, NodeRegistry, TaskRouter (Wave 7)
  • ✓ SSH transport for remote node commands
  • ✓ Subagent spawning and lifecycle management
  • ○ Fleet-aware experiment node placement
  • ○ Cross-machine cargo workspace evaluation
  • ○ Binary rollback via ripcord (safe experiment failure recovery)
Planned

Local Model Integration

Run experiments with local models for zero-cost, zero-latency, fully private autonomous improvement. BitNet 1-bit models can run 100B parameters on a single CPU.

  • ○ BitNet provider (1-bit LLM inference on CPU)
  • ○ Local model download and setup via wizard
  • ○ Hardware-aware kernel selection (ARM NEON / x86 AVX)
  • ○ Continuous local experimentation (zero API cost)
Planned

Self-Training Pipeline

The recursive loop: Fawx runs experiments to generate training data, fine-tunes its own local model on the winners, and uses the improved model for future experiments.

  • ○ Training data curation from winning experiment patches
  • ○ LoRA fine-tuning orchestration
  • ○ Model evaluation as a fitness criterion
  • ○ Signed adapter deployment via consensus chain
